[GCP VPC network best practices](https://cloud.google.com/solutions/best-practices-vpc-design):

- [Use cutom mode subnets](https://cloud.google.com/solutions/best-practices-vpc-design#custom-mode)
- [Use fewer large subnets rather than many small subnets](https://cloud.google.com/solutions/best-practices-vpc-design#fewer-subnets)
- [Use a single VPC network per GCP project](https://cloud.google.com/solutions/best-practices-vpc-design#vpc-per-project)
- [Use fewer large firewall rules and organise them hierarchically](https://cloud.google.com/solutions/best-practices-vpc-design#hybrid_design_connecting_an_on-premises_environment)
- [Use service accounts rather than network tags on VM instances for controlling firewall rules](https://cloud.google.com/solutions/best-practices-vpc-design#isolate-vms-service-accounts)
